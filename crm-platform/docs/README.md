# ドキュメント一覧

このディレクトリには、統合CRMプラットフォームの詳細なドキュメントが含まれています。

## ドキュメント構成

### 📘 [実装ドキュメント](IMPLEMENTATION.md)

実装の詳細を記録したドキュメントです。

**内容**:
- プロジェクト構造
- データベーススキーマの詳細
- 主要機能の実装
- ユーティリティ関数
- 設定ファイル
- 変更履歴

**更新タイミング**: 実装に変更が入るたびに更新

---

### 📊 [スキーマリファレンス](SCHEMA_REFERENCE.md)

データベーススキーマの完全なリファレンスです。

**内容**:
- Enum型定義
- テーブル定義（全カラムの詳細）
- リレーション定義
- インデックス定義

**更新タイミング**: スキーマに変更が入るたびに更新

---

### 📝 [変更履歴](CHANGELOG.md)

プロジェクトのすべての重要な変更を記録します。

**内容**:
- バージョンごとの変更内容
- 追加・変更・削除された機能
- セキュリティ修正

**更新タイミング**: 変更が入るたびに `[Unreleased]` セクションに追加

---

## ドキュメント更新チェックリスト

実装を変更した際は、以下のチェックリストを確認してください:

### スキーマ変更時

- [ ] `SCHEMA_REFERENCE.md` の該当テーブル/Enum型を更新
- [ ] `IMPLEMENTATION.md` の「データベーススキーマ」セクションを更新
- [ ] `CHANGELOG.md` の `[Unreleased]` セクションに変更内容を追加

### 新機能追加時

- [ ] `IMPLEMENTATION.md` の「主要機能の実装」セクションに追加
- [ ] 使用例を含める
- [ ] `CHANGELOG.md` の `[Unreleased]` セクションに追加

### プロジェクト構造変更時

- [ ] `IMPLEMENTATION.md` の「プロジェクト構造」セクションを更新
- [ ] 新しいディレクトリ/ファイルの説明を追加

### 設定ファイル変更時

- [ ] `IMPLEMENTATION.md` の「設定ファイル」セクションを更新
- [ ] 変更内容と理由を説明

### リリース時

- [ ] `CHANGELOG.md` の `[Unreleased]` を新しいバージョン番号のセクションに移動
- [ ] 日付を更新
- [ ] すべてのドキュメントの「最終更新日」を更新

---

## ドキュメントの目的

これらのドキュメントは、以下の目的で作成されています:

1. **開発者の理解を助ける**: 実装の詳細を理解しやすくする
2. **保守性の向上**: 変更履歴を記録し、将来の変更を容易にする
3. **オンボーディング**: 新しいメンバーがプロジェクトに参加しやすくする
4. **設計の記録**: 設計判断の理由を記録し、将来の参考にする
